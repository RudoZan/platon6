🔍 En revisión

El proceso de desarrollo de software

Ahora que has construido y publicado tu primer recurso digital, has entrado oficialmente al mundo de la creación de software. Aunque programes con la ayuda de una inteligencia artificial y no escribas todo el código manualmente, es muy importante que conozcas las estrategias y etapas que hacen que un recurso sea confiable, fácil de usar y duradero.

1. Planificación y diseño de la interfaz

Antes de abrir VS Code o pedirle código a la IA, dedica unos minutos a planificar tu recurso en papel o mediante un esquema mental simple. Define con claridad lo siguiente:

  • El flujo de la actividad: ¿Qué verá el estudiante primero? ¿Qué pasa si responde correctamente? ¿Qué pasa si falla?
  • Las entradas y salidas: Qué datos ingresará el usuario (números, texto, opciones) y qué resultados mostrará la pantalla.
  • La distribución visual: Ubicar dónde irá el enunciado, las ayudas visuales y los botones de acción para que el estudiante no se distraiga ni se confunda.

2. Estrategias de prueba (Testing)

La IA suele escribir código funcional muy rápido, pero también puede introducir errores sutiles. Para asegurar la calidad de tu recurso sin ser un experto técnico, debes guiarte por tres niveles de prueba:

Pruebas Unitarias (Verificar piezas individuales): Consiste en aislar y probar una función específica del recurso. Por ejemplo, si tienes una fórmula matemática para validar una respuesta, pruébala ingresando datos correctos e incorrectos para ver si el resultado parcial es el esperado.

Pruebas de Integración (Verificar el flujo conjunto): Consiste en probar que las distintas partes del recurso conversen bien entre sí. Por ejemplo, verificar que al presionar el botón de enviar respuesta se actualice el contador de puntaje, se oculte la pregunta actual y se muestre la tarjeta de explicación.

Pruebas de Regresión (Asegurar que nada se rompió): Es muy común que al corregir un error o agregar una nueva función (como un botón de pista) se rompa algo que ya funcionaba bien antes. Siempre que la IA haga un cambio en tu código, vuelve a probar el recurso completo desde el inicio para verificar que las funciones antiguas siguen intactas.

3. Usabilidad, accesibilidad y documentación

Un buen recurso educativo digital debe poder usarse sin explicaciones externas adicionales. Considera estos tres aspectos en el diseño:

  • Instrucciones claras: El recurso debe guiar al estudiante de manera intuitiva. Incluye ayudas visuales sencillas o textos cortos que expliquen el formato de respuesta esperado (por ejemplo, si se debe responder con coma o con un formato algebraico específico).
  • Mensajes de error informativos: Si el estudiante comete un error de formato, el sistema no debe congelarse. Debe mostrar una alerta clara que le explique qué ocurrió y cómo corregirlo.
  • Facilidad de navegación: Evita pantallas recargadas de información. La autoevaluación, los botones de ayuda y las explicaciones deben aparecer de forma organizada para no provocar fatiga cognitiva.

4. Optimización y rendimiento del recurso (Consideraciones No-Código)

En el desarrollo de software y páginas web, **no todo es código**. Una aplicación puede tener una arquitectura perfecta en JavaScript, pero si sus recursos multimedia son pesados o no están optimizados, la experiencia del usuario colapsará. Los estudiantes suelen acceder a los recursos desde redes escolares lentas, conexiones móviles Wi-Fi saturadas o dispositivos de gama baja.

Por ello, la gestión eficiente del **peso de los archivos** y la elección del **formato correcto** es una disciplina clave en el desarrollo web profesional:

  • El costo invisible del ancho de banda: Si una página web carga 10 imágenes de 8 MB cada una (80 MB en total), un teléfono celular en red 3G/4G tardará más de un minuto en abrirla o consumirá el plan de datos del estudiante. Optimizar cada archivo al mínimo necesario es un estándar de accesibilidad.
  • Resolución versus Visualización: Nunca subas una foto de cámara de 4000x3000 px si en la pantalla solo se mostrará en un recuadro de 800x600 px. Redimensionar las dimensiones en píxeles antes de guardar ahorra hasta un 90% de peso sin perder nitidez.
  • Limpieza de código y activos: Pídele a la IA que elimine fragmentos de código de prueba o estilos repetidos, y elimina siempre de tus carpetas los archivos gráficos de respaldo o versiones anteriores que ya no utilice tu aplicación web.

Guía y comparación de los 4 grandes formatos de imagen en la Web

Elegir el formato de imagen adecuado según el tipo de contenido (foto, ilustración, animación o icono con transparencia) es fundamental para lograr un sitio veloz y nítido:

Formato ¿Soporta Transparencia? ¿Soporta Animación? Características, Ventajas y Cuándo Usar
WebP ✅ Sí (Canal Alfa) ✅ Sí (WebP Animado) El estándar web moderno de Google. Combina lo mejor de todos: ofrece una compresión fotográfica superior (pesa ~30% menos que JPG y hasta ~90% menos que PNG) manteniendo una nitidez excepcional y fondo transparente. Compatible con todos los navegadores modernos.
👉 Cuándo usar: Es la primera opción para prácticamente todo en la web actual (fotos, mapas, iconos recortados y animaciones).
JPG / JPEG ❌ No (Fondo blanco) ❌ No El formato clásico para fotografías. Utiliza compresión "con pérdida" (elimina detalles imperceptibles al ojo) para lograr archivos muy pequeños.
👉 Cuándo usar: Fotografías realistas, retratos y fondos donde NO necesites transparencia ni recortes.
PNG ✅ Sí (Canal Alfa perfecto) ❌ No (excepto APNG) Formato sin pérdida (Lossless). Guarda cada píxel exactamente como fue creado sin comprimir colores, logrando bordes ultra nítidos y transparencias suaves. Sin embargo, genera archivos sumamente pesados cuando se aplica a fotografías o mapas detallados.
👉 Cuándo usar: Logotipos, diagramas, capturas de pantalla de texto o gráficos vectoriales donde la extrema nitidez sea indispensable y no se pueda usar WebP o SVG.
GIF ⚠️ Sí (Pero solo de 1 bit / sin bordes suaves) ✅ Sí (El formato clásico animado) El veterano de las animaciones simples. Está limitado a una paleta de máximo 256 colores, por lo que las fotos se ven con "puntitos" (dithering) y las transparencias dejan bordes dentados o blancos. Además, sus archivos suelen ser muy pesados en comparación con un video o un WebP animado.
👉 Cuándo usar: Memes rápidos, pequeños iconos animados simples o retro-compatibilidad extrema donde no se admita MP4 o WebP animado.
El rol del creador de recursos

Al programar con inteligencia artificial, tu rol principal deja de ser el de transcriptor de código y pasa a ser el de diseñador de la experiencia de aprendizaje y supervisor de calidad. Tu criterio pedagógico y tus pruebas minuciosas son las que transforman un trozo de código generado en una verdadera herramienta educativa.
